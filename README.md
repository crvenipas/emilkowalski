# Состав и динамическая эволюция семейства астероидов Emilkowalski

Репозиторий содержит Jupiter-ноутбуки, реализующие двухэтапный анализ молодого семейства астероидов Emilkowalski: уточнение его численного состава и реконструкцию каскадного сценария разрушения родительского тела (14627).

Работа выполнена в рамках дипломной работы специалиста по специальности 03.05.01 «Астрономия» на кафедре астрономии, геодезии, экологии и мониторинга окружающей среды Уральского федерального университета (2026).

## Ноутбуки

### `family_membership.ipynb` — Уточнение состава семейства

Реализует полный конвейер выделения и верификации членов семейства Emilkowalski по каталогам собственных и оскулирующих элементов орбит (AstDyS-2).

**Методы:**
- Иерархическая кластеризация (HCM) с квази-метрикой *d* (орбитальное расстояние, м/с)
- Метрика Холшевникова ρ₅ в пространстве собственных элементов орбит
- Критерий *K/L* (статистическое расширение HCM)
- Дополнительная проверка по метрике ρ₂ для оскулирующих орбит

**Входные данные:**
- `proper_catalog24.dat` — каталог собственных элементов орбит AstDyS-2 (~1.25 млн объектов)
- Каталог оскулирующих элементов орбит AstDyS-2

**Алгоритм:**
1. Фильтрация области интереса (ROI): ±0.01 по *a*, *e*, sin *i* вокруг тела (14627) → 183 объекта
2. Кластеризация по диапазонам порогов: *d* ∈ [3, 10] м/с, ρ₅ ∈ [3×10⁻⁴, 7×10⁻⁴]
3. Выделение устойчивого ядра семейства как пересечения кластеров при фиксированных порогах (*d*_cut = 5.0 м/с, ρ₅,cut = 5×10⁻⁴)
4. Проверка кандидатов по критерию *K/L* и метрике ρ₂

**Результаты:**
- **Строгий состав (HCM):** 17 объектов — 13 известных членов + 4 новых кандидата (2009UL13, 2019VE44, 2020UZ20, 2021TU55)
- **Расширенный состав (K/L):** 18 объектов — дополнительно 2014WE584
- Precision: 76.47% / Recall: 100% (HCM); Precision: 72.22% / Recall: 100% (K/L)

### `cascade-reconstruction.ipynb` — Реконструкция хронологии каскадного распада

Строит хронологическое дерево разрушения родительского тела на основе результатов численного интегрирования орбит на интервале 5 млн лет (OrbFit/Orbit9) с учётом эффекта Ярковского.

**Входные данные:**
- Файлы интегрирования орбит в формате `vast_kep_xyz_{id}.txt` — по каждому объекту и каждому сценарию дрейфа большой полуоси (директории `nominal/`, `N/`, `S/`, `P/`)
- Список членов семейства из `family_membership.ipynb`

**Этапы алгоритма:**

| Этап | Описание |
|------|----------|
| 0 | Формирование выборки и задание сценариев осей Ярковского (5 фиксированных тел + кандидатные оси N/S/P) |
| 1–2 | Перебор всех пар (784 комбинации для 17 объектов) и загрузка данных интегрирования |
| 3 | Глобальное согласование осей: итеративное распространение от 5 опорных тел |
| 4 | Оценка пар: вычисление *T*ρ₂, *T*_{ΔΩ,Δω}, ρ₂,min, *f*_min для каждой пары |
| 5 | Кластеризация пар по времени *T*ρ₂ → эпохи распада |
| 6 | Построение дерева каскада (граф от корня 14627) и визуализация |
| 7 | Выгрузка таблиц |

**Ключевые параметры:**

| Параметр | Значение по умолчанию | Описание |
|----------|-----------------------|----------|
| `RHO2_MAX` | 0.05 | Порог метрики Холшевникова ρ₂ |
| `F_MIN_MAX` | 0.4° | Порог сглаженного *f*(−*T*) (по Pravec et al. 2018) |
| `DELTA_T_MAX` | 0.1 Млн лет | Допустимое расхождение *T*ρ₂ и *T*_{ΔΩ,Δω} |
| `EPS_T` | 0.15 Млн лет | Ширина окна кластеризации эпох |
| `T_MAX` | 1.80 Млн лет | Физический потолок возраста семейства |

**Выходные файлы** (суффикс `_17` или `_18` — размер выборки):

| Файл | Раздел диплома | Содержание |
|------|----------------|------------|
| `pair_summary_{N}.csv` | 4.2 | Все надёжные пары: *T*ρ₂, *T*_{ΔΩ,Δω}, ΔT, ρ₂,min, *f*_min, флаг надёжности |
| `axis_assignment_{N}.csv` | 4.4 | Назначенный сценарий оси Ярковского для каждого объекта |
| `cascade_tree_{N}.csv` | 4.4 | Рёбра дерева каскада с временами и эпохами |
| `epochs_summary_{N}.csv` | 4.3–4.4 | Сводка по эпохам разрушения |
| `cascade_tree_v2_{N}.png` | 4.4 | Визуализация дерева на временно́й шкале |
